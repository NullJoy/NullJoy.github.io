AI Infrastructure & ML Systems Overview

2025-04-28 Tech Misc AI infrastructure, Career development, ML systems Comments Word Count: 390(words) Read Count: 2(minutes)

Evolution of AI Infrastructure

Stage 1: 2016-2017 - MXNet Era

  • AWS MXNet (later donated to Apache)
  • DeepEngine
  • Amazon SageMaker

Stage 2: Meta PyTorch Revolution

  • E.g. PyTorch Accelerator enablement: Model deployment and application enablement
  • State-of-the-art (SOTA) implementations
  • CUDA kernel optimization
  • Compiler advancements

Stage 3: Hardware Optimization

  • Focus on GPU efficiency
  • Cost reduction in training

AI Infrastructure in Engineering

Both involves Research & Development.

Framework

  • ML System
  • Hardware cost optimization

Platform

  • Inference platform
  • Training platform
  • Cluster management

AI Infrastructure Engineer’s Scope

Core Objectives

  • Making AGI development more cost-effective and faster
  • Hardware-software co-design
  • System architecture (OSDI, NSDI)
  • Model training and inference expertise
  • Transformer architecture
  • Generative AI systems

Current Applications

  • Enterprise solutions (RAG, agents)
  • The “One-person Billion-dollar Company” concept

Career Development Paths

For New Graduates

  1. Education

    • AI and ML system courses
    • Online ML systems courses

  2. Project Experience

    • GenAI projects
      • Infrastructure track: Performance metrics (latency improvement, throughput supported)
      • Research track: Algorithm improvements, hardware co-design

  3. Open Source Contributions

    • Popular projects (PyTorch, vLLM, SGLang)
    • Focus on throughput and latency optimization

For 3-5 Years Experience

  • Strategic career planning
  • Transition opportunities:
    • Backend/database ->
    • AI platform, infrastructure and serving (AI storage solutions, etc.) ->
    • Career move
  • Current advantage: Emerging field with equal opportunity
  • Continuous learning and adaptation
  • Emphasis on design and implementation

For AI Infrastructure Managers

  • Domain expertise requirements
  • Focus areas:
    • 20-30% design review
    • People management
    • Less hands-on coding

Learning Resources

Educational Content

  • AI System School (Uncle Moon)
  • Andrej Karpathy’s YouTube series:
    • Basic MLP implementation
    • Transformer model development
    • Model training techniques

Academic Programs

  • Berkeley/CMU ML Systems courses
  • Stanford ML Systems webinars (weekly)

Community Resources

  • CUDA/GPU programming communities
  • Technical meetups
  • Open source project participation

Career Transition Preparation

Key Areas

  1. Technical Skills

    • LeetCode practice
    • System design study
    • Technical blog reading

  2. Timing Strategy

    • Monitor headcount availability
      • Headcounts are usually out after yearly planning
    • Early application preparation
      • Don’t wait, start interviews process asap
    • Proactive networking

Open Source Projects in AI Infrastructure

Training and Serving

  1. vLLM
  2. SGLang

AI Native Cloud Platforms

  • Enhanced observability
  • Improved debugging capabilities

Agent Systems

  • Inter-agent coordination
  • Plug-and-play memory systems
  • Context fetching optimization

Startup Landscape

Pre-LLM Era

  • Databricks (Spark-based)
  • Anyscale (Ray-based)
  • Focus on compute, storage, networking

Current Landscape

  • Model serving infrastructure
  • Abstraction layers for different model providers
  • Enable users to quick build a compound AI system
  • Notable companies:
    • Fireworks AI
    • Lepton
    • Together AI
